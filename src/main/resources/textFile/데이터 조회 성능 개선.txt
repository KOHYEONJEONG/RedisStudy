그중 sql 튜닝을 제일 먼저 해야한다.

1. SQL 튜닝은 기존의 시스템 변경 없이 성능을 개선할 수 있다.
2. 근본적인 문제를 해결하는 방법으로 SQL 튜닝 가능성이 높다.
SQL 자체가 비효율적으로 작성됐다면 아무리 시스템적으로 성능을 개선한다고 하더라도 한계가 있다.

문제) 성능이 부족하니 레디스를 붙히면 된다?
ㄴ no, sql에서 내가 비효율적으로 혹시나 뭔가 작성되어 있는 부분이 없는지를 먼저 확인

* 요약하자면 db 성능 개선 방법들 중 가장 가성비가 좋은 방법이 SQL 튜닝이다.

[결론]
캐싱은 추가적인 시스템 도입과 비용이 필요.
SQL 튜닝은 시스템 변경 없이 성능을 개선하고 근본 원인을 해결하는 첫 번째 단계입니다.
